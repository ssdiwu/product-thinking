## 这个技能是什么？

<img width="1024" height="572" alt="image" src="https://github.com/user-attachments/assets/864c45b9-6045-4189-98fd-12f4d96ea3ac" />

**产品思维顾问**是一个帮助你的产品想法落地、或诊断已有产品问题的 AI 助手。
通过七大思维工具进行系统性分析，输出可执行的产品策略文档。

**核心能力**：
- ✅ 快速验证产品想法的可行性
- ✅ 深入理解用户真实需求
- ✅ 设计最小可行产品（MVP）
- ✅ 诊断已有产品的问题
- ✅ 提供产品优化方向

## 它能解决什么问题？

### 新产品/创意阶段

| 你的困惑 | 技能如何帮助 |
|----------|--------------|
| 有个想法，不知道是否靠谱 | 通过【灵魂三问】快速验证核心价值，避免伪需求 |
| 功能越想越多，不知从何下手 | 用【减法思维】聚焦 MVP，用最小成本验证假设 |
| 不确定用户真正需要什么 | 通过【JTBD】和【故事思维】挖掘用户真实需求 |
| 担心产品会失败 | 用【逆向思维】进行事前验尸，提前识别风险 |
| 想不到应用场景 | 用【场景应用】工具找到合适的落地场景 |

### 已有产品阶段

| 你的困惑 | 技能如何帮助 |
|----------|--------------|
| 产品做出来了，但没有用户 | 通过【灵魂三问】重新审视定位，用【逆向思维】诊断失败原因 |
| 数据不好，不知道哪里有问题 | 分析用户旅程，找到流失节点和优化机会 |
| 功能堆砌，用户反馈复杂 | 用【减法思维】砍掉非核心功能，聚焦核心价值 |
| 增长遇到瓶颈 | 重新评估产品定位，找到新的增长方向 |

## 如何使用？
### 快速开始（3 分钟体验）

<img width="1024" height="572" alt="image" src="https://github.com/user-attachments/assets/87821fe8-3a4b-46f6-9ab7-c3469abd1782" />

**第一步：说出你的想法或问题**

```
我想做一个帮助程序员记录代码笔记的工具
```

或者：

```
我的记账 App 用户留存很低，怎么办？
```

**第二步：回答"灵魂三问"**

AI 会问你三个核心问题（或从你的描述中提取）：

| 问题 | 示例回答 |
|------|----------|
| 谁是用户？ | 正在学习新技术的程序员，经常需要记录代码片段 |
| 痛点是什么？ | 现有笔记工具不支持代码高亮，搜索不方便 |
| 为什么选择你？ | 我自己是程序员，深知这个痛点，且有技术积累 |

**第三步：获得分析报告**

AI 会输出结构化的分析文档，包含：
- 用户画像
- MVP 方案
- 风险评估
- 下一步行动

### 深度分析（完整体验）

<img width="1024" height="572" alt="image" src="https://github.com/user-attachments/assets/193edf41-a131-4a3f-9e18-6cbc61742303" />

AI 会根据你的需求，选择合适的分析工具：

```
你的问题 → AI 自动选择工具 → 产出分析报告
```

#### 工具选择逻辑

| 你说... | AI 使用... | 你会得到... |
|---------|-----------|------------|
| "这是不是伪需求？" | **问题发现** | 问题真实性验证、五个发现渠道 |
| "用户真正需要什么？" | **JTBD + 故事思维** | 用户三层任务（功能/情感/社会）、三维画像、旅程地图 |
| "这样做会不会失败？" | **逆向思维（事前验尸）** | 潜在失败原因列表、针对性预防措施 |
| "功能太多不知道砍哪个？" | **减法思维（MVP）** | 核心假设清单、最小功能集、功能优先级 |
| "想不到应用场景？" | **场景应用** | 四大落地场景建议（数据分析/自动化/个人/家庭） |
| "快速判断一下" | **灵魂三问** | 快速评估结论 |

### 已有产品诊断示例

```
用户：我的记账 App 用户留存很低，怎么办？

AI 分析流程：

1. 灵魂三问（重新审视定位）
   - 用户是谁？→ 刚工作的年轻人，记账意识不强
   - 痛点是什么？→ 记账太麻烦，容易忘记
   - 优势是什么？→ 界面简洁，但没有突出价值

2. 逆向思维（假设产品失败，倒推原因）
   - 失败原因1：用户没有持续记账的动力
   - 失败原因2：手动录入太繁琐
   - 失败原因3：记账后没有获得正向反馈

3. 用户旅程（找到流失节点）
   - 注册→首次记账→坚持一周→流失（节点在第二周）

4. 输出优化建议
   - 加入自动记账功能（降低门槛）
   - 添加储蓄目标可视化（增加动力）
   - 设计周报月报（提供正向反馈）
```

## 你会得到什么？

### 输出物示例

#### 1. 灵魂三问快速评估

```markdown
## 用户画像
- 目标用户：25-35 岁的程序员，正在学习新技术
- 使用场景：在看教程、读博客时需要记录代码片段
- 痛点强度：中等（现有方案能用但不爽）

## 痛点验证
- 当前方案：用 Notion/Typora 记录
- 痛点：不支持代码高亮、搜索不智能、无法按语言分类
- 用户意愿：愿意尝试更好的工具

## 独特优势
- 技术积累：熟悉代码解析和高亮
- 用户洞察：自身就是目标用户
- 差异化：专注代码笔记场景，而非通用笔记
```

#### 2. JTBD 深度分析

```markdown
## 功能层任务
- 快速记录代码片段
- 支持 20+ 主流编程语言语法高亮
- 智能搜索代码内容

## 情感层任务
- 感觉自己是个有条理的程序员
- 减少重复造轮子的焦虑
- 学习新技术的成就感

## 社会层任务
- 向同事展示专业的代码管理
- 在技术社区分享时更有说服力
```

#### 3. 事前验尸分析

```markdown
## 假设 6 个月后产品失败，可能的原因：

| 风险类别 | 失败原因 | 预防措施 | 优先级 |
|----------|----------|----------|--------|
| 需求风险 | 程序员更习惯用 GitHub Gist | 强调本地快速记录的差异化 | 高 |
| 产品风险 | 代码高亮支持的语言不够多 | 支持自定义 TextMate 语法 | 中 |
| 推广风险 | 找不到目标用户聚集地 | 在技术教程网站投放广告 | 高 |
| 竞争风险 | Notion/Obsidian 发布代码块增强 | 加深垂直领域的专业功能 | 中 |
```

#### 4. MVP 设计

```markdown
## 核心假设
1. 程序员需要专门的代码笔记工具
2. 语法高亮 + 智能搜索是核心价值
3. 本地存储比云同步更重要（隐私）

## MVP 功能（绝不砍）
- [x] 代码片段快速录入
- [x] 自动语言识别 + 语法高亮
- [x] 全文搜索代码内容
- [x] 按语言/标签分类

## 砍掉的功能（后续迭代）
- 云同步（v1.1）
- 代码分享（v1.2）
- Markdown 编辑器（v2.0）
- 移动端 App（v2.0）
```

## 最佳实践

### ✅ 推荐做法

| 做法 | 原因 |
|------|------|
| **大胆说出想法** | 哪怕不成熟，技能会帮你梳理逻辑 |
| **准备回答问题** | AI 会问"灵魂三问"，尽量具体描述 |
| **保持开放心态** | 事前验尸可能会戳中痛点，但能帮你避坑 |
| **参考真实案例** | 技能内置多个案例分析，可以要求查看 |
| **关注可执行性** | 分析报告的核心是"下一步做什么" |

### ❌ 避免误区

| 误区 | 正确认知 |
|------|----------|
| "我的想法很完美" | 保持怀疑，技能帮你找盲点 |
| "功能越多越好" | MVP 思维会帮你聚焦核心价值 |
| "所有人都是用户" | 需要精准定位，避免泛化 |
| "大厂没做肯定没市场" | 大厂看不上的可能是你的机会 |
| "分析完就万事大吉" | 分析只是起点，市场验证才是答案 |

## 常见问题

**Q: 这个技能会帮我写代码吗？**

A: 不会。产品思维技能专注于产品策略分析。技术实现建议使用其他技能（如 coding 相关）。

---

**Q: 分析结果一定对吗？**

A: 不一定。分析是基于框架的系统性思考，帮你减少盲点，但市场验证才是最终答案。这个技能的作用是**提高成功率**，而非**保证成功**。

---

**Q: 可以分析任何类型的产品吗？**

A: 大部分可以。包括 Web 应用、移动 App、SaaS、工具类产品等。纯硬件/实体产品可能需要额外考虑供应链、制造成本等因素。

---

**Q: 我已经有产品了，还能用吗？**

A: 可以。技能支持已有产品的诊断、优化方向评估、功能取舍决策。事实上，已有产品的分析往往更有价值，因为你有真实数据可以参考。

---

**Q: 需要我提供哪些信息？**

A: 最少只需要：
- 你想做什么产品（或现有产品是什么）
- 你认为的目标用户是谁

更多信息（如现有数据、竞品情况）会让分析更精准，但不是必须的。

---

**Q: 分析需要多长时间？**

A: 取决于分析深度：
- 快速评估（灵魂三问）：2-3 分钟
- 完整分析（多工具组合）：5-10 分钟
