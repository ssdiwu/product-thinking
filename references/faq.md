# 常见问题与最佳实践

## 最佳实践

### ✅ 推荐做法

| 做法 | 原因 |
|------|------|
| **大胆说出想法** | 哪怕不成熟，技能会帮你梳理逻辑 |
| **准备回答问题** | AI 会问"灵魂三问"，尽量具体描述 |
| **保持开放心态** | 事前验尸可能会戳中痛点，但能帮你避坑 |
| **参考真实案例** | 技能内置多个案例分析，可以要求查看 |
| **关注可执行性** | 分析报告的核心是"下一步做什么" |

### ❌ 避免误区

| 误区 | 正确认知 |
|------|----------|
| "我的想法很完美" | 保持怀疑，技能帮你找盲点 |
| "功能越多越好" | MVP 思维会帮你聚焦核心价值 |
| "所有人都是用户" | 需要精准定位，避免泛化 |
| "大厂没做肯定没市场" | 大厂看不上的可能是你的机会 |
| "分析完就万事大吉" | 分析只是起点，市场验证才是答案 |

## 常见问题

**Q: 这个技能会帮我写代码吗？**

A: 不会。产品思维技能专注于产品策略分析。技术实现建议使用其他技能（如 coding 相关）。

---

**Q: 分析结果一定对吗？**

A: 不一定。分析是基于框架的系统性思考，帮你减少盲点，但市场验证才是最终答案。这个技能的作用是**提高成功率**，而非**保证成功**。

---

**Q: 可以分析任何类型的产品吗？**

A: 大部分可以。包括 Web 应用、移动 App、SaaS、工具类产品等。纯硬件/实体产品可能需要额外考虑供应链、制造成本等因素。

---

**Q: 我已经有产品了，还能用吗？**

A: 可以。技能支持已有产品的诊断、优化方向评估、功能取舍决策。事实上，已有产品的分析往往更有价值，因为你有真实数据可以参考。

---

**Q: 需要我提供哪些信息？**

A: 最少只需要：
- 你想做什么产品（或现有产品是什么）
- 你认为的目标用户是谁

更多信息（如现有数据、竞品情况）会让分析更精准，但不是必须的。

---

**Q: multi-agent 模式和 codex-assisted 模式有什么区别？**

A:
- **multi-agent**: AI 自动派发专家 Agent 并行执行，适合快速迭代
- **codex-assisted**: AI 生成任务清单，用户用 Codex 执行，适合精确控制

选择建议：需要快速迭代 → multi-agent；需要精确控制 → codex-assisted

---

**Q: 如何调整并发数？**

A: 并发数支持三种配置方式（按优先级从高到低）：

1. 启动参数：`/product-design --concurrency 5`
2. `project.yaml` 配置：`concurrency: 5`
3. 默认值：3

配置建议：Claude Pro 用户建议 3-5，企业用户可设置 5-10，测试环境建议 2。
