# 我要做一个什么样的产品？

> 用户：小水

## 产品创意来源

### 1. 从自身痛点出发

**方法**：
- 记录烦恼日记
- 留意自己抱怨的时刻
- 思考"为什么没有更好的方案"

**案例**：
- 自己总是忘记回复消息 → 做一个消息提醒工具
- 自己记不住看过什么文章 → 做一个稍后阅读工具

### 2. 从他人抱怨出发

**方法**：
- 关注社交媒体上的抱怨
- 留意朋友/同事的吐槽
- 参与相关社区/论坛

**案例**：
- 朋友抱怨记账太麻烦 → 思考简化记账流程
- 同事抱怨会议太多 → 思考会议效率工具

### 3. 从竞品不足出发

**方法**：
- 使用现有产品
- 找到不满意的地方
- 思考如何改进

**案例**：
- 现有工具功能太多 → 做一个极简版
- 现有工具价格太贵 → 做一个平价版
- 现有工具界面丑 → 做一个美观版

## 产品创意验证

### 第一步：灵魂三问

1. **谁是用户？** → 能说出具体用户的名字/画像
2. **痛点是什么？** → 能说出具体痛点场景
3. **为什么是你？** → 能说出自己的独特优势

如果三个问题都能清晰回答，继续下一步。

### 第二步：和潜在用户聊

**聊什么**：
- 确认问题真实存在
- 了解当前解决方案
- 询问愿意尝试新方案吗

**注意**：
- 不要推销
- 多听少说
- 保持开放

### 第三步：MVP 验证

**做什么**：
- 最小可行产品
- 能验证核心假设
- 快速上线测试

## 我的案例

### 案例 1：稍后阅读工具

**来源**：自己总是忘记看收藏的文章

**灵魂三问**：
1. 谁是用户？→ 自己，以及有同样习惯的人
2. 痛点是什么？→ 收藏但不看，越积越多
3. 为什么是你？→ 我自己就是用户

**MVP**：
- 一个输入框，输入链接
- 一个列表，显示所有稍后阅读
- 一个完成按钮

**结果**：自己每天在用

### 案例 2：极简待办清单

**来源**：现有待办APP功能太多

**灵魂三问**：
1. 谁是用户？→ 喜欢简洁的人
2. 痛点是什么？→ 功能太多，太复杂
3. 为什么是你？→ 我自己就用极简风格

**MVP**：
- 添加任务
- 完成任务
- 没有其他功能

**结果**：几个朋友也在用

## 心得体会

1. **从小处开始**：不要一开始就想做大产品
2. **自己是第一个用户**：自己用的产品最真实
3. **快速验证**：不要埋头太久，早点让用户用
