# AI 编程脚手架案例分析

## 背景

作者想做一个 AI 编程工具的脚手架模板，帮助用户快速启动项目。

## 灵魂三问分析

### 1. 谁是用户？
- 使用 Claude Code/AI 编程工具的用户
- 想快速启动项目，不想从零搭建
- 用户可能是独立开发者、创业者、学生

### 2. 痛点是什么？
- 每次启动新项目都要重复搭建基础结构
- 不知道用什么技术栈
- 搭建过程中容易踩坑、浪费时间

### 3. 为什么选择你？
- 作者自己就是用户，深刻理解痛点
- 有丰富的 AI 编程经验
- 能持续更新模板

## JTBD 分析

### 功能层任务
快速获得一个可运行的项目基础结构

### 情感层任务
感到"终于可以开始做真正的东西了"

### 社会层任务
显得专业、高效

## 逆向思维（事前验尸）

### 可能的失败原因
1. 技术栈更新快，模板跟不上
2. 用户需求多样化，一个模板无法满足
3. 推广渠道有限，找不到用户

### 预防措施
1. 模块化设计，方便快速更新
2. 支持多种技术栈变体
3. 在 AI 编程社区推广

## MVP 设计

### 核心假设
1. 用户需要脚手架模板
2. 用户愿意使用 CLI 工具生成项目
3. 用户愿意为此贡献/付费

### MVP 功能
- [ ] 基础项目结构
- [ ] 配置文件模板
- [ ] 快速开始文档
- [ ] CLI 生成工具

### 砍掉的功能
- 完整的 UI 组件库（后续迭代）
- 复杂的状态管理（后续迭代）
- 部署配置（后续迭代）

## 场景定位

**自动化脚本场景**：用模板替代重复性搭建工作

## 关键经验

1. **自己就是用户**：作者自己就用这个模板，确保产品解决真实问题
2. **社区驱动**：鼓励用户贡献，保持模板更新
3. **简单优先**：MVP 只包含最基础的结构，后续按需扩展
