# 灵魂三问

## 这个框架什么时候用

当你需要快速判断一个产品方向是否值得继续深入时，用灵魂三问。

它适合的场景：

- 新想法刚冒出来，想先判断方向是否站得住
- 说得出很多功能，但说不清用户、痛点和优势
- 想在进入 JTBD、MVP 或更深分析前，先做一次快速门控
- 用户描述很热闹，但你怀疑核心前提还没站稳

它的作用不是产出漂亮定位语，而是快速识别三个最核心的前提有没有基本成立：

1. 谁是用户
2. 痛点是什么
3. 为什么由你来做

## 什么时候别用

以下情况不要把灵魂三问当成主框架：

- 已经进入已有产品诊断，问题在留存、转化或链路异常
- 当前最主要的问题是“需求是不是真的存在”
- 你已经明确知道用户和痛点，真正卡住的是 MVP 收敛
- 用户需要的不是方向判断，而是完整用户任务拆解

这时应优先考虑：

- 问题发现
- 逆向思维
- JTBD
- MVP / 减法思维

## 先问哪几个高判别问题

默认只问少量问题，不要把三问扩展成问卷。

### 1. 谁是用户

要问的不是“人群标签”，而是：

- 你能不能说出一个具体用户，而不是一类泛人群？
- 你见过他在什么场景下遇到这个问题？
- 你是否接触过这个用户，还是只是想象他存在？

### 2. 痛点是什么

要问的不是“你想做什么功能”，而是：

- 用户现在怎么解决这个问题？
- 当前方案最不爽的点是什么？
- 这个问题出现得多频繁、多痛、多值得切换？

### 3. 为什么由你来做

要问的不是“你准备做得多好”，而是：

- 你有什么独特资源、视角或切口？
- 你为什么比更大的玩家更容易先做对这件事？
- 如果别人也能做，你的起跑线到底是什么？

## 常见红旗

### 用户侧红旗

- “所有人都是用户”
- 只有年龄、职业这类泛画像，没有真实场景
- 说不出任何一个具体用户
- 没接触过用户，只是凭感觉定义用户

### 痛点侧红旗

- 把“想要某功能”当成痛点
- 说不清当前替代方案
- 只说“不方便”，说不出成本、频率或失败后果
- 用户并没有积极找替代方案

### 优势侧红旗

- “界面更好看”
- “功能更强”
- “我们会更努力”
- 说不出为什么大厂没做、没做好，或者为什么你更适合先做

## 什么时候该停

出现以下情况时，不要继续往下包装结论，应先停下来补依据：

- 谁是用户说不清
- 当前替代方案说不清
- 痛点只有抽象形容，没有场景和行为
- 为什么由你来做说不清
- 三问中有任一问只能靠“我觉得”支撑

停下后的动作：

- 去找真实用户
- 补一个具体场景
- 补当前替代方案和痛点证据
- 先做问题发现，而不是继续讲价值主张

## 最小验证动作

灵魂三问通过与否，不靠主观自信判断，至少做一个最小验证。

优先验证动作：

1. 找 3 到 5 个目标用户，复述你的三问答案，看他们是否认同
2. 让用户描述自己现在怎么解决，而不是让他评价你的想法
3. 把“为什么由你来做”改写成一句可检验的话，看是否真的有资源或优势支撑

最小成功信号：

- 能说出 1 到 3 个真实用户或真实用户样本
- 能描述当前替代方案和具体痛点
- 能明确说出一个别人不容易复制的切口

最小失败信号：

- 只能复述抽象画像
- 痛点停留在“可能有需求”
- 所谓优势只是主观自信或一般性能力

## 输出时怎么写

不要把三问写成长解释，优先写成判别结果。

推荐结构：

```markdown
## 灵魂三问判断

### 现象
- 当前用户描述：
- 当前痛点描述：
- 当前优势描述：

### 判断
- 谁是用户：已知 / 推断 / 待验证
- 痛点是否成立：已知 / 推断 / 待验证
- 为什么由你来做：已知 / 推断 / 待验证

### 红旗
- [最关键的 1-3 个红旗]

### 动作
- [下一步最值得做的动作]

### 最小验证
- [找谁验证]
- [验证什么]
- [成功 / 失败信号]
```

## 一个短例子

```markdown
### 现象
- 用户被描述为“独立开发者”
- 痛点被描述为“管理反馈太乱”
- 优势被描述为“我自己也做产品”

### 判断
- 谁是用户：仍然偏泛，需要收窄到具体类型，例如“正在做付费 SaaS 的独立开发者”
- 痛点是否成立：方向可能成立，但还没说清当前替代方案和混乱成本
- 为什么由你来做：只有身份认同，还没有独特切口

### 红旗
- 用户范围过大
- 痛点只有感觉，没有行为证据
- 优势没有明确资源支撑

### 动作
- 找 3 个独立开发者访谈他们现在如何收集和处理反馈

### 最小验证
- 成功信号：至少 2 个用户明确表达当前流程混乱且已在找替代方案
- 失败信号：用户虽然觉得麻烦，但并不愿意切换现有做法
```
